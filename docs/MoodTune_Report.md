## Abstract

MoodTune is a multimodal deep learning system that bridges computer vision and generative music. A user's facial expression is captured via webcam, classified into one of seven emotional categories by a fine-tuned EfficientNet-B0 convolutional neural network, and then used to drive two complementary outputs: (1) an original ~3-minute melody generated by a two-layer LSTM conditioned on the detected emotion, and (2) a curated list of songs selected to improve the user's emotional state. The system achieves approximately **72% accuracy** on the FER2013 benchmark — above the human agreement ceiling of ~65% — using advanced training techniques including two-phase transfer learning, Mixup augmentation, label smoothing, Test-Time Augmentation (TTA), and CosineAnnealing scheduling. All components are integrated into an interactive Streamlit web application.

## 1. Introduction

### 1.1 Problem Statement

Emotional well-being is closely tied to music. Research in music psychology consistently shows that musical properties — tempo, mode (major vs. minor), dynamics, rhythmic density — correspond systematically to emotional states (Juslin & Sloboda, 2010). However, selecting music that matches or therapeutically counters one's mood typically requires conscious effort and self-awareness.

This project asks: **can a deep learning system automatically detect a person's emotional state from their face and respond with original generated music and curated recommendations designed to support their well-being?**

### 1.2 Motivation

- **Facial expressions** are among the most direct, universally recognized signals of human emotion, making them a natural input modality for an emotion-aware system.
- **Transfer learning** has made high-accuracy facial expression recognition accessible without massive datasets or compute budgets.
- **Neural sequence models** (LSTMs) can learn stylistic patterns from music corpora and generate new sequences conditioned on external signals such as emotion.
- The combination of **perception (vision) → generation (audio) → recommendation** showcases a complete multimodal AI pipeline, demonstrating how multiple deep learning components can be composed into a real application.

### 1.3 Goals

1. Train a CNN-based emotion classifier achieving >70% accuracy on FER2013 using advanced fine-tuning techniques.
2. Train an LSTM melody generator conditioned on detected emotion.
3. Map emotions to musically grounded parameters (tempo, scale, dynamics) and structure generated melodies into song-length (~3 min) compositions.
4. Recommend existing songs that shift the user's mood in a positive direction.
5. Deliver all components through a polished, interactive browser-based interface.

## 3. Techniques and Methods

### 3.1 Facial Expression Recognition

#### 3.1.1 The FER2013 Challenge

FER2013 is a widely used benchmark for facial expression recognition. It contains 35,887 labeled 48×48 grayscale images across 7 emotion classes. Key challenges include:

- **Label noise**: FER2013 was collected with automatic labeling. Human agreement on the labels is only ~65%, meaning some images are genuinely ambiguous or mislabeled. This acts as a hard ceiling on achievable accuracy.
- **Low resolution**: 48×48 pixels provides limited facial detail. We upsample to 224×224 for EfficientNet-B0's native resolution.
- **Class imbalance**: The "disgusted" class has ~436 samples vs. 7,215 for "happy."

#### 3.1.2 EfficientNet-B0 Architecture

We use **EfficientNet-B0** as the backbone, upgraded from an initial ResNet-18 baseline (~65-68% accuracy).

EfficientNet (Tan & Le, 2019) introduces **compound scaling** — a principled method to simultaneously scale network width, depth, and resolution by a fixed ratio. EfficientNet-B0 is the baseline model in this family, offering an excellent accuracy/efficiency tradeoff:

- Backbone: MBConv blocks (Mobile Inverted Bottleneck with Squeeze-and-Excitation)
- Parameters: ~4M (vs. 11M for ResNet-18) with better representational power
- Native input: 224×224

**Modifications for FER2013:**

1. **Grayscale to 3-channel**: FER2013 images are single-channel. We repeat the grayscale channel 3 times — `x.repeat(1, 3, 1, 1)` — preserving all pretrained weights without modifying the first conv layer.

2. **Classifier replacement**: The original 1000-class ImageNet head is replaced with:
   ```python
   Sequential(Dropout(p=0.3), Linear(1280, 7))
   ```
   Dropout before the final layer adds regularization, addressing the tendency to overfit on FER2013's noisy labels.

**Accuracy vs. baseline:**

| Model | Input | Accuracy |
|---|---|---|
| ResNet-18 (v1 baseline) | 48×48 | ~65-68% |
| EfficientNet-B0 | 96×96 | ~68.5% |
| EfficientNet-B0 + all techniques | 224×224 | **~72%** |

#### 3.1.3 Two-Phase Transfer Learning

Rather than fine-tuning all layers simultaneously from the start, we use a **two-phase training strategy**:

**Phase 1 — Frozen backbone (5 epochs, LR=1e-3):**
- All EfficientNet backbone weights are frozen; only the new classifier is trained.
- Purpose: Rapidly train the new classifier head without corrupting pretrained features.
- Higher learning rate is safe because only 645K parameters are updated.

**Phase 2 — Full fine-tuning (up to 95 epochs, LR=1e-4):**
- All layers unfrozen; the entire network is fine-tuned end-to-end.
- Lower learning rate prevents catastrophic forgetting of ImageNet features.
- The model refines its feature representations specifically for facial expressions.

This approach consistently outperforms either freezing or full fine-tuning alone, as the classifier learns a good initialization before the backbone adapts.

#### 3.1.4 Label Smoothing

FER2013's label noise means that assigning 100% confidence to any single label may be wrong. **Label smoothing** (Szegedy et al., 2016) addresses this by replacing hard one-hot targets with soft distributions:

```
y_smooth = (1 - ε) × y_onehot + ε / K
```

With `ε = 0.1` and `K = 7` classes, the correct class gets `0.914` probability and each other class gets `0.014`. This prevents the model from becoming overconfident and improves calibration and generalization on noisy labels.

#### 3.1.5 Mixup Augmentation

**Mixup** (Zhang et al., 2018) is a data augmentation technique that trains on convex combinations of training samples:

```
x̃ = λ·xᵢ + (1−λ)·xⱼ
ỹ = λ·yᵢ + (1−λ)·yⱼ
```

Where `λ ~ Beta(α, α)` with `α = 0.2`. Applied with 50% probability per batch, Mixup:
- Forces the model to learn linear behavior between classes
- Acts as a strong regularizer against memorizing noisy labels
- Reduces the gap between training and validation accuracy

#### 3.1.6 Data Augmentation Pipeline

```python
transforms.Compose([
    transforms.Grayscale(num_output_channels=1),
    transforms.Resize((224, 224)),
    transforms.RandomHorizontalFlip(),           # Faces are roughly symmetric
    transforms.RandomRotation(15),               # Handle head tilt
    transforms.RandomAffine(translate=(0.1,0.1), # Handle position variation
                            scale=(0.9, 1.1)),
    transforms.ColorJitter(brightness=0.3,       # Handle lighting changes
                           contrast=0.2),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.5], std=[0.5]),
    transforms.RandomErasing(p=0.25,             # Simulate occlusions (glasses, etc.)
                             scale=(0.02, 0.15)),
])
```

**RandomErasing** is particularly useful for FER2013, where users may have glasses, hair, or partial occlusions covering parts of their face.

#### 3.1.7 Class Imbalance Handling

FER2013's disgust class has only 436 training samples vs. 7,215 for happy — a 16.5× imbalance. We compute **inverse-frequency class weights**:

```
wc = N_total / (K × Nc)
```

These weights are passed to `CrossEntropyLoss`, penalizing the model more heavily for misclassifying rare emotions. This prevents the model from ignoring minority classes.

#### 3.1.8 CosineAnnealingWarmRestarts Scheduler

Instead of a fixed learning rate or simple step decay, we use **CosineAnnealingWarmRestarts** (Loshchilov & Hutter, 2017):

```
lr(t) = lr_min + 0.5(lr_max - lr_min)(1 + cos(π × t/T₀))
```

The learning rate follows a cosine curve and **restarts** periodically with `T_0=10, T_mult=2`. This allows the model to:
- Escape local minima via periodic LR increases (warm restarts)
- Converge to sharper minima via the cosine cooldown
- Explore different loss landscape regions across restarts

#### 3.1.9 Gradient Clipping

To stabilize training (especially during Phase 2 when all layers are updated), we apply gradient clipping:

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

This caps the global gradient norm, preventing exploding gradients when the backbone is unfrozen.

#### 3.1.10 Test-Time Augmentation (TTA)

At inference time, we average predictions from two forward passes:
1. Original face crop
2. Horizontally flipped face crop

```python
logits = (model(face) + model(flip(face))) / 2
```

TTA consistently improves accuracy by 0.5–1.5% by reducing variance in the prediction, particularly for expressions like anger/disgust that can appear on either side of the face.

|---|---|

| Sad accuracy | ~72% |

Fearful and surprised are commonly confused because both involve wide eyes and raised eyebrows. The model's ~72% accuracy exceeds the estimated human agreement on FER2013 (~65%), which represents the dataset's label quality ceiling.

### 3.2 Face Detection with Haar Cascades

Before emotion classification, the face must be located within the webcam frame.

**Haar Cascade Classifier** (Viola & Jones, 2001) is a classical machine learning approach:

1. **Haar-like features**: Rectangular filters computing pixel-sum differences between adjacent regions — capturing edges, lines, and contrast. Fast to compute via integral images (O(1) per region).
2. **AdaBoost ensemble**: Hundreds of weak classifiers (single Haar features) combined into a strong classifier.
3. **Cascade architecture**: Classifiers arranged in stages of increasing complexity. Easy "non-face" rejections happen in early stages, so only promising regions reach expensive later stages — enabling near-real-time detection.

**Configuration:**
- Cascade: `haarcascade_frontalface_default.xml` (OpenCV built-in)
- Scale factor: 1.3 (multi-scale pyramid)
- Min neighbors: 5 (reduces false positives)
- Selection: Largest detected face by area

**Preprocessing after detection:**
1. Crop the face region from the grayscale frame
2. Resize to 224×224 (EfficientNet-B0 native resolution)
3. Normalize: mean=0.5, std=0.5
4. Add batch dimension → shape: `(1, 1, 224, 224)`

### 3.3 LSTM Melody Generation

#### 3.3.1 Architecture

The melody generator uses a **two-layer LSTM** (Long Short-Term Memory) conditioned on the detected emotion:

```
MelodyLSTM Architecture
=======================

Input at each timestep t:
  note_embedding(nt)    →  64-dim
  emotion_embedding(e)  →  64-dim (same at every step)
  concatenated input    → 128-dim

LSTM layers:
  LSTM Layer 1: 128 → 256 hidden units
  Dropout (0.3)
  LSTM Layer 2: 256 → 256 hidden units

Output projection:
  FC(256 → 128): Linear → vocab size (MIDI notes 0-127)
  Softmax → note probability distribution
```

The key architectural choice is **concatenating the emotion embedding at every timestep** rather than only at initialization. This continuously conditions the generation, preventing the emotion signal from fading over long sequences.

**Total parameters: ~500K** — lightweight enough to generate 300-note sequences in under 2 seconds on CPU.

#### 3.3.2 Why LSTM for Music?

Music is a **sequential art form** — each note's appropriateness depends on what came before. LSTM cells address the vanishing gradient problem in vanilla RNNs through **gating mechanisms**:

- **Forget gate**: Decides what past information to discard
- **Input gate**: Decides what new information to store
- **Output gate**: Controls what information to expose

This allows the model to capture long-range dependencies — for example, learning to resolve phrases back to the tonic after melodic tension builds over many notes.

#### 3.3.3 Training

- **Data**: Synthetic sequences generated by a rule-based algorithm with emotion-specific melodic biases (700 sequences, directional patterns per emotion — e.g., happy melodies tend upward, sad melodies tend downward).
- **Objective**: Next-note prediction (cross-entropy loss, `ignore_index=0` for padding/rests).
- **Sequence length**: 64 notes per training sample, sliding window with 50% overlap.
- **Optimizer**: Adam, LR=1e-3, weight decay=1e-5.
- **Scheduler**: ReduceLROnPlateau (patience=10).
- **Early stopping**: patience=20, stopped at epoch 45.
- **Device**: MPS (Apple Metal Performance Shaders).

#### 3.3.4 Generation with Scale Masking and Temperature

At inference, the LSTM generates autoregressively:

```
For step t = 1..N:
  1. Embed current note + emotion
  2. LSTM forward pass → logit vector (128 dims)
  3. Apply scale mask: set logits of out-of-key notes to -inf
  4. Divide logits by temperature T
  5. Softmax → in-scale probability distribution
  6. Sample next note from distribution
```

**Scale masking** ensures every generated note belongs to the emotion's musical scale, preventing dissonant "wrong notes." This is a hard constraint imposed at generation time without retraining.

**Temperature** controls creativity:
- T < 1.0: More conservative, stays near likely notes
- T = 1.0: Faithful to learned distribution
- T > 1.0: More random, more surprising choices

We use T=0.8 for a good balance of musicality and variety.

#### 3.3.5 Song Structure

Rather than generating a flat sequence of notes, the generator creates **structured compositions** of approximately 3 minutes:

```
Song Structure
==============
Intro    →  Verse 1  →  Chorus 1  →  Verse 2 (variation)
         →  Chorus 2  →  Bridge   →  Outro (tonic resolution)
```

Each section uses a distinct LSTM-generated phrase. Repeated sections include small rhythmic and dynamic variations (different velocity offsets, occasional duration changes) to prevent mechanical repetition — mimicking how human musicians "comp" the same phrase differently across passes.

### 3.4 Emotion-to-Music Parameter Mapping

The bridge between perception and generation is a hand-crafted mapping grounded in **music psychology research**:

| Emotion | Tempo (BPM) | Scale | Octave | Note Density | Velocity |
|---|---|---|---|---|---|
| Happy | 120 | Major | 5 | 80% | 100 |
| Sad | 60 | Minor | 4 | 40% | 60 |
| Angry | 140 | Minor | 4 | 90% | 120 |
| Fearful | 90 | Minor | 4 | 60% | 50 |
| Surprised | 130 | Major | 5 | 70% | 90 |
| Neutral | 100 | Major | 4 | 50% | 80 |
| Disgusted | 80 | Minor | 3 | 50% | 70 |

**Rationale from music theory:**
- **Tempo**: Faster tempos feel energetic and agitated; slower feel contemplative and melancholic.
- **Scale/Mode**: Major scales sound bright; minor scales sound dark. One of the most robust emotion-music associations.
- **Octave**: Higher pitches feel lighter; lower pitches feel heavier.
- **Note density**: Dense passages feel energetic or anxious; sparse passages feel calm.
- **Velocity (dynamics)**: Loud notes convey intensity; soft notes convey gentleness.

### 3.5 Digital Audio Synthesis

Generated melodies are rendered directly to audio — no external MIDI synthesizer or soundfont required.

#### 3.5.1 MIDI-to-Frequency Conversion

Equal temperament formula:

```
f = 440 × 2^((n - 69) / 12)
```

Where n is the MIDI note number. A4 (MIDI 69) = 440 Hz. Each semitone = 2^(1/12) ≈ 1.0595× the previous.

#### 3.5.2 Additive Harmonic Synthesis with Vibrato

A pure sine wave sounds thin and artificial. We mix four partials:

```
signal(t) = 0.55 × sin(2π × f(t) × t)     — fundamental
           + 0.25 × sin(2π × 2f(t) × t)   — 2nd harmonic (octave)
           + 0.12 × sin(2π × 3f(t) × t)   — 3rd harmonic (perfect 5th)
           + 0.08 × sin(2π × 4f(t) × t)   — 4th harmonic
```

Where `f(t) = freq × (1 + 0.003 × sin(2π × 5t))` introduces subtle **vibrato** (5 Hz, ±0.3% pitch variation), adding warmth and naturalness to long-held notes.

The relative amplitudes of harmonics determine **timbre** — this configuration approximates a mellow piano/bell hybrid, which works well for melodic material across all 7 emotions.

#### 3.5.3 Full ADSR Envelope

Each note is shaped by a four-stage amplitude envelope:

```
Attack:   Linear ramp from 0 to 1 over 30ms
Decay:    Linear fade from 1 to sustain level over 100ms
Sustain:  Held at 70% amplitude for the body of the note
Release:  Linear fade to 0 over min(150ms, 20% of note duration)
```

This produces a piano-like quality — quick onset, slight decay, sustained body, clean release. Without this envelope, note transitions would produce audible clicks (Gibbs phenomenon) from abrupt amplitude changes.

#### 3.5.4 Audio Pipeline

```
Note sequence → per-note waveform (ADSR + harmonics)
             → concatenate all segments
             → peak-normalize to 80% amplitude
             → quantize: float64 → int16 (16-bit PCM)
             → write WAV at 44,100 Hz sample rate
             → BytesIO buffer → st.audio()
```

### 3.6 Content-Based Song Recommendation

#### 3.6.1 Mood Improvement Strategy

Rather than recommending music that matches the detected emotion (which might reinforce sadness with sad songs), MoodTune uses a **mood-improvement** strategy:

| Detected Emotion | Strategy | Target Valence | Target Energy |
|---|---|---|---|
| Sad | Uplifting | 0.6–1.0 | 0.4–0.7 |
| Angry | Calming | 0.5–0.8 | 0.2–0.5 |
| Fearful | Reassuring | 0.6–0.9 | 0.3–0.6 |
| Disgusted | Refreshing | 0.5–0.8 | 0.4–0.7 |
| Happy | Reinforce | 0.7–1.0 | 0.6–1.0 |
| Surprised | Grounding | 0.5–0.8 | 0.4–0.7 |
| Neutral | Energizing | 0.5–0.8 | 0.4–0.7 |

#### 3.6.2 Audio Features

Songs are characterized by two features (Spotify audio analysis conventions):

- **Valence (0–1)**: Musical positiveness. High = happy/euphoric. Low = sad/angry.
- **Energy (0–1)**: Perceptual intensity. High = loud/fast/dense. Low = soft/mellow.

#### 3.6.3 Filtering Algorithm

```
1. Look up target valence/energy ranges for detected emotion
2. Filter songs.csv: valence ∈ [v_min, v_max] AND energy ∈ [e_min, e_max]
3. If < N matches: relax both ranges by ±0.1, retry
4. If still < N: return random sample from full catalog
5. Randomly sample N songs and return with Spotify links
```

This is **content-based filtering** — recommendations depend entirely on item attributes, not user history, making it effective for a cold-start single-user scenario.

Key observations from live webcam testing:

- **Happy** is the most reliably detected emotion (large smile → distinctive)

- **Glasses** reduce accuracy (shadows/reflections near eyes confuse the model)
- **Lighting** quality is the biggest practical factor — well-lit frontal faces work well
- Fixing the **label-order mismatch** (ImageFolder sorts alphabetically: neutral=4, sad=5, surprised=6) was critical — the initial deployment was mapping all predictions incorrectly for 3 of 7 classes

### 6.2 Key Engineering Decisions

**Why EfficientNet-B0 over ResNet-18?**
EfficientNet-B0 achieves better accuracy per parameter via compound scaling and MBConv blocks with squeeze-and-excitation. On FER2013 with the same training setup, it improved accuracy from ~68% to ~72%.

**Why 224×224 instead of native 48×48?**
EfficientNet-B0 was pretrained on 224×224 images. Feeding 48×48 (or even 96×96) misaligns the spatial feature map sizes the model expects, limiting the benefit of ImageNet pretraining. Upsampling to 224 gives access to the full pretrained feature hierarchy.

**Why LSTM over Transformer for melody?**
For sequences of ~64-300 notes on a small synthetic dataset, LSTMs are more data-efficient. Transformers require much more data to learn useful attention patterns and are harder to condition on external signals (emotion) at every step. With only 700 training sequences, LSTM was the appropriate choice.

**Why sine synthesis over MIDI/soundfont?**
Sine-wave synthesis with harmonics requires zero external dependencies (no FluidSynth, no soundfont files) and produces consistent cross-platform audio. The ADSR envelope and harmonic mixing provide sufficient tonal quality for a demonstration system.

**Why content-based over collaborative filtering?**
Content-based filtering works with zero user history — appropriate for a single-session app where no user data is available. It also allows the mood-improvement logic to be directly encoded as feature range constraints.

## 8. Discussion

### 8.1 Limitations

1. **FER2013 label noise ceiling**: ~72% is near the practical maximum for this dataset without dataset cleaning. Switching to AffectNet (~450K cleaner images) would likely push accuracy to 80%+.

2. **Glasses and occlusion**: Haar Cascade + EfficientNet both struggle with partial face occlusions. A modern face detector (MTCNN, RetinaFace) and training data with more occlusion examples would help.

3. **LSTM on synthetic data**: The LSTM was trained on synthetically generated music rather than real MIDI files. While the scale masking ensures musical correctness, the melodies lack the expressive phrasing of human-composed music. Training on a real MIDI corpus (e.g., MAESTRO dataset) would produce more nuanced output.

4. **Single-frame prediction**: Emotion is inferred from a single photo. Video-based emotion recognition (averaging predictions over frames) would be more robust to momentary expression changes.

### 8.2 Future Work

- **AffectNet dataset**: ~450K images with cleaned annotations, expected to push accuracy to 80-85%
- **Video-based FER**: Temporal CNN or transformer over video frames
- **Real MIDI training data**: MAESTRO or Lakh MIDI dataset for higher-quality melody generation
- **Chord generation**: Add a second LSTM to generate harmonization alongside the melody
- **Personalization**: Track user feedback on recommendations to build a user preference model
